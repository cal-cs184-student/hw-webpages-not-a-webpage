Overview

In this homework, we implement a cloth simulation system using a mass-spring model.

We first implement a basic simulation of a cloth-like object using a mass-spring system, where each vertex is represented as a point mass connected by three types of springs (structural, shearing and bending).
We then implement a numerical integration method to simulate the motion of the cloth over time, taking into account forces such as gravity and damping.
We also implement a collision detection and response system to handle interactions between the cloth and other objects in the scene, such as a sphere or a plane.
Then, we implement a self-collision detection system to handle cases where the cloth collides with itself. We use a hash table to efficiently detect self-collisions with vertex pairs that are close to each other.
Finally, we implement a shader program to render the cloth with various shading techniques, including diffuse, Phong shading, texture mapping, bump mapping, displacement mapping and mirror-like reflections.

3. Section III: Handling collisions with other objects

  • Implementation:

    • Sphere: I use the sphere's center and radius to check if the distance between the center and the vertex is smaller than the radius. If so, the tangent point is calculated by the following formula:
      tangent=origin+(dir/dist)∗radius;tangent = origin + (dir / dist) * radius;tangent=origin+(dir/dist)∗radius;
      where origin is the vertex position, dir is the direction from the center to the vertex, and dist is the distance between the center and the vertex. The new position of the vertex is then calculated by:
      position=last_position+(1−friction)∗(tangent−last_position)position = last\_position + (1 - friction) * (tangent - last\_position)position=last_position+(1−friction)∗(tangent−last_position)

    • Plane: I determine whether a PointMass has crossed a plane by checking if the sign of its signed distance changes. The signed distance is calculated following the formula:
      signed_distance=(last_position−point)∗normalsigned\_distance = (last\_position - point) * normalsigned_distance=(last_position−point)∗normal

    If the sign changes, I calculate the rough tangent point by the following formula:

    tangent=last_position+t∗(position−last_position);tangent = last\_position + t * (position - last\_position);tangent=last_position+t∗(position−last_position);

4. Section IV: Handling Self-Collisions

  • Implementation:

To efficiently detect self-collisions, we use a hashtable to reduce the computational complexity from O(n2)O(n^2)O(n2) to O(n)O(n)O(n). This ensures that only nearby PointMass objects are checked for potential collisions.

The hashtable is built based on the positions of the PointMass objects using the hash_position() function. For example, along the x-axis:

float w = 5 * width / num_width_points;
int bx = floor(pos.x / w);

The build_hash_table() function iterates through all PointMass objects, computes their hash values using hash_position(), and stores them in the hashtable.

Then, in the self_collide() function, for a given PointMass pmpmpm, we only check collisions with others in the same hash bucket. If the distance between two masses is smaller than twice the fabric's thickness, a collision is detected. A correction is calculated and averaged over all collisions:

correction /= (count * simulation_steps);
pm.position += correction;

In simulate(), after calling build_hash_table(), we call self_collide() for each PointMass.

5. Section V: Shaders

  • Explanation:

A shader program runs on the GPU and is composed of two main components: the vertex shader and the fragment shader.
The vertex shader handles per-vertex operations such as transforming 3D coordinates and computing normals, while the fragment shader operates on rasterized fragments (similar to pixels), computing lighting and color effects for each one.

For instance, in the mirror shader, the vertex shader computes world-space positions and normals. The fragment shader then uses these to reflect the view direction and sample from the environment map to produce realistic mirror-like reflections. Because shaders execute in parallel on the GPU, they allow real-time rendering performance, in contrast to CPU-based ray tracing.

5.4 Bump & Displacement Mapping

Bump mapping perturbs the surface normal using a height map to simulate surface detail without modifying geometry.

Displacement mapping, on the other hand, alters the vertex positions using the height map, creating actual geometry changes.
