Overview

This projects devides into two parts: Bezier Curves and Surfaces & Triangle Meshes and Half-Edge Data Structure. These two parts are generally intended for rendering complex shapes.
For the first part, I implemented the de Casteljau Algorithm to generate Bezier Curves and Surfaces. It enables me to create curves & surfaces out of moving control points.
For the second part, I used the half-edge data structure and implemented the area-weighted vertex normals, edge flip, edge split, and Loop Subdivision for mesh upsampling. These methods together provide strong support for editing meshes and upsampling.

1.Section I: Bezier Curves and Surfaces

1.1. Bezier Curves

  • Implementation:
    The idea of de Casteljau Algorithm is to generate a weighted sum of the control points with given weight t, so that the new control points can be used to recursively to generate the next level of control points. Since for each step, the number of control points is reduced by 1, the algorithm will terminate when the number of control points is 1. The final point will move along a curve when t changes from 0 to 1, which is the Bezier curve.

1.2. Bezier Surfaces

  • Implementation:
    The idea of Bezier Surfaces is to generate a 2D surface with 2 parameters u and v. The algorithm extends the original by taking multiple parallel sets of control points that generate a curve into account. For a shared u, the curves generate a new set of control points that lies on a same vertical plane, therefore can generate a new curve with the parameter v. Since u is continuous, the curves can be expanded to a surface.

2. Section II: Triangle Meshes and Half-Edge Data Structure

2.1. Area-Weighted Vertex Normals

  • Implementation:
    To calculate the area-weighted vertex normals, I implemented the following 2 functions:

    • area(FaceCIter f): This function takes an FaceCIter as input and returns the area of the face. It's implemented by calculating the half-edge vectors and taking the cross product of them.
    • normal( void ): It iters through every halfedge by visiting h = h->twin()->next() until it meets the original halfedge. For each halfedge, it gets its corresponding face and calls area()

  • Results


    As can be seen above, the Phong shading benefits from calculating the Specular Shading using the area-weighted vertex normals. It outperforms the flat shading by providing a more realistic and smooth surface.

2.2. Edge Flip

  • Implementation:
    The implementation of the edge flip is straightforward. For a non-boundary edge, I get its corresponding halfedges, exaustively visited the halfedges' member variables and set them to the corresponding halfedges, vertices, edges and faces. Finally I exaustively visited the other halfedges, faces and vertices for variable reassignment.

    Despite its straightforward essence, coding for it is a bit tricy due to the complexity of the half-edge data structure. I've implemented it seemingly correctly, only to find that there will be missing faces for multiple edge flips.

    A trick for it is to prepare an exaustive list of elements that you need to visit and update, and then carefully update them one by one even if you consider it unneccessary. The above bug is caused by that I considered it unnessary to update the face() for some edges, however it turns out that it'll result in missing faces.

2.3 Edge Split

  • Implementation:
    The implementation of splitEdge is all about creating new vertices, edges, and faces, and assign each of their member variables to the corresponding element in the split version of the mesh. To be more specific, I first declarated all the new elements and assigned existing vertices, faces, edges and halfedges to variables, so that I can easily access them. Then I followed the edge-vertex-face sequence to assign values to these variables for the simple bottom-to-up philosophy. Similar to the debug trick in the previous task, I exaustively visited all the elements that need to be updated, and carefully updated them one by one.

    Luckily, thanks to the my debug trick leant from the previous task, my realization of the edge split is correct by one-shot and I didn't encounter any bugs during the implementation.

2.4 Loop Subdivision for Mesh Upsampling

  • Implementation:
    The implementation of Loop Subdivision is by far the most exausting one over all tasks. For now, I'll only introduce how I implement the origin Loop Subdivision algorithm, and I'll introduce the modified version the Results session.

    The original Loop Subdivision algorithm is implemented by the following steps:

    • preprocess: Before the start of each function call, I visited the vertices and edges to set their isNew to false, to clean up the mess from the previous iteration.
    • setNewPositions: I first Iter through every vertex in the mesh, compute new positions according to the Loop Subdivision and store them in the Vertex::newPosition member variable.
    • setEdgePosition: I implemented setEdgePosition and called it for each edge the set the positions for upcoming new vertices.
    • subdivide: I split every edge in the mesh calling splitEdge. I also made minor change to the implementation of splitEdge to record isNew for new vertices and edges.
    • FlipEdge: I iter through all the edges, flipped any new edge that connects an old and new vertex.
    • UpdatePosition: At last, I visited all the vertices and updated their positions to the new positions.
